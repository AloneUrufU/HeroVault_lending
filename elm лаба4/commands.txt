PowerShell — команды для подготовки окружения (выполнить в PowerShell, при необходимости — от имени администратора)

# 1) Установить Node.js (через winget; если нет — установите вручную)
winget install OpenJS.NodeJS.LTS -e

# 2) Установить Elm и инструменты
npm install -g elm elm-format @elm-tooling/elm-language-server

# 3) Установить расширение Elm LS для VS Code (если есть команда code)
code --install-extension elmtooling.elm-ls-vscode

# 4) Проверить установки
node -v
npm -v
elm --version
elm-format --version
elm-language-server --version

Инструкция запуска Elm-проекта и отладки (Windows)

1) Перейдите в папку проекта
   cd "c:\Users\User Andrew\Desktop\elm"

2) Проверьте наличие исходного модуля Main.elm в src
   dir src\*.elm

   Если файл называется main.elm — переименуйте:
   # PowerShell
   Rename-Item src\main.elm -NewName Main.elm

3) Скомпилируйте Elm в JS (создаст src\elm.js)
   elm make src/Main.elm --output=src/elm.js

4) Проверьте, что elm.js создан:
   dir src\elm.js

5) Запустите локальный сервер:
   elm reactor
   или (альтернатива)
   python -m http.server 8000
   или
   npx http-server -c-1 -p 8000

   Откройте в браузере:
   http://localhost:8000/src/index.html
   или (для elm reactor)
   http://localhost:8000/src/index.html

6) При обновлении .elm:
   - Каждый раз после изменения запускайте:
     elm make src/Main.elm --output=src/elm.js
   - Обновляйте страницу в браузере: Ctrl+F5 или включите в DevTools Disable cache

7) Устранение типичных проблем:
   - Если в консоли: "Не удалось загрузить elm.js" — проверьте, существует ли src\elm.js и правильный ли путь.
   - Если "Elm is not defined" — возможно elm.js не скомпилирован или загружен другой файл; выполните шаг 3 и убедитесь, что скрипт загружается без кэша (см. cache-bust в index.html).
   - Откройте DevTools → Network, обновите страницу и посмотрите запрос к elm.js (должен вернуть 200 и реальный размер файла).

Конец инструкции.
